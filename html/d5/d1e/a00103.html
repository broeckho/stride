ContactType::IdSubscriptArray<unsigned int> stride::Population::m_currentContactPoolId
private

The contact pool counters (one per type of pool) for assigning pool IDs.

Counters generate a non zero UID that's unique per type of pool, so <type, UID> uniquely detemines the pool. UID zero means 'NA" e.g. wor[lace UID for a K12school student wiil be zero. As a defensive measure, the ContactPoolSys gets initialized with (for each type) an empty pool in the vecotor storing the contact pools. As a consequence, one has: if UID != 0 then ContactPoolsSys[type][UID].GetId() == UID for all type the index in the vector with pools is identical to the pool's UID.
